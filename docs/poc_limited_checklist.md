# 限定的PoC実施チェックリスト（Iteration-07）

**作成日**: 2026-03-16
**バージョン**: 1.0
**目的**: システムの動作確認とハルシネーション検証（統計的精度検証ではない）

---

## 1. 限定的PoCの目的

### 1.1 本チェックリストの位置づけ

**重要**: 本チェックリストは「統計的精度検証」ではなく、「動作確認」を目的としています。

- ✅ **目的**: システムが正常に動作し、ハルシネーションがないことを確認
- ❌ **目的外**: 相関係数、偽陽性率、偽陰性率の統計的測定

### 1.2 統計的精度検証との違い

| 項目 | 限定的PoC（本チェックリスト） | 統計的精度検証（Phase 3） |
|---|---|---|
| **サンプル数** | 1件（実動画1件のみ） | 5〜10件 |
| **人間評価** | なし | あり（比較用） |
| **測定指標** | 動作確認、ハルシネーション | 相関係数、偽陽性率、偽陰性率 |
| **実施時期** | Phase 2完了後（運用開始前） | Phase 3（運用開始後） |
| **合格基準** | システム動作OK、警告0件 | 相関 > 0.7、FPR < 20%、FNR < 10% |

### 1.3 実施タイミング

**Phase 2完了後、運用開始前**に実施します。

```
Phase 2 (開発)
  ↓
Iteration-07（本チェックリストの実施）
  ↓
運用開始
  ↓
Phase 3（統計的精度検証）
```

## 3. 実施手順（ステップバイステップ）

### Step 1: 動画ファイルの準備

- [ ] 実動画ファイルを用意（過去の面談動画1件）
- [ ] ファイル形式を確認（MP4, MOV, AVI, WebM）
- [ ] ファイルサイズを確認（500MB以下推奨）
- [ ] 動画時間を確認（10〜30分程度）
- [ ] 画質・音質を確認（表情・音声が明瞭）

**動画の選定基準**:
- 標準的な面談動画（特に問題なく、スムーズに進行）
- 候補者の表情・視線が明瞭に映っている
- 音声がクリアで聞き取りやすい

### Step 2: Streamlit UIでの解析

#### 2-1: UIの起動

- [ ] ブラウザで `http://localhost:8501` を開く
- [ ] UIが正常に表示されることを確認

#### 2-2: APIキーの入力

- [ ] サイドバーの「Gemini APIキー」欄にAPIキーを入力
- [ ] 「使用モデル」で「gemini-2.5-flash」を選択

#### 2-3: 動画のアップロード

- [ ] 「面談動画をアップロード」ボタンをクリック
- [ ] 動画ファイルを選択
- [ ] ファイル情報（ファイル名、サイズ、形式）が正しく表示されることを確認

#### 2-4: 解析の実行

- [ ] 「解析開始」ボタンをクリック
- [ ] 「動画を解析中...」のメッセージが表示されることを確認
- [ ] エラーなく解析が完了することを確認（数分〜10分）

### Step 3: 結果の確認

#### 3-1: 総合評価

- [ ] 総合スコアが表示されることを確認（0〜100）
- [ ] リスクレベルが表示されることを確認（低/中/高）

#### 3-2: 詳細評価

- [ ] 4つのカテゴリ（コミュニケーション、ストレス耐性、信頼性、協調性）が表示されることを確認
- [ ] 各カテゴリにスコア、観察事実、確信度が含まれることを確認

#### 3-3: リスクシグナル・ポジティブシグナル

- [ ] リスクシグナルのセクションが表示されることを確認
- [ ] ポジティブシグナルのセクションが表示されることを確認

#### 3-4: 推奨事項

- [ ] 推奨事項が表示されることを確認

#### 3-5: JSON出力

- [ ] 「詳細なJSON出力を表示」を展開
- [ ] JSON形式で評価結果が表示されることを確認
- [ ] 「JSONをダウンロード」ボタンでJSONファイルがダウンロードできることを確認

## 4. 確認項目（品質チェック）

### 4.1 システム動作確認

- [ ] **動画アップロード**: エラーなくアップロードできた
- [ ] **AI解析**: エラーなく解析が完了した
- [ ] **結果表示**: 評価レポートが正常に表示された
- [ ] **JSON出力**: JSONファイルがダウンロードできた

**判定**: 全てチェックできた場合、システム動作は **OK**

### 4.2 出典品質確認

- [ ] **観察事実に出典が含まれているか**: 各観察事実の末尾に「(参照: ナレッジベース)」が含まれる
- [ ] **出典の妥当性**: 出典が適切なセクション名を指している（例: 非言語コミュニケーション基準）

**確認方法**:
```json
// JSON出力の例
"observations": [
  "適切なアイコンタクトを維持（参照: 非言語コミュニケーション基準）",  // ✅ 出典あり
  "明瞭で論理的な発話（参照: 言語表現評価基準）"  // ✅ 出典あり
]
```

**判定**: 全ての観察事実に出典が含まれている場合、出典品質は **OK**

### 4.3 ハルシネーション検証

- [ ] **出典欠落の警告**: 出典が欠落している観察事実がないか
- [ ] **確信度低の警告**: 確信度「低」が3つ以上あるか
- [ ] **矛盾検出の警告**: スコア80以上で否定的キーワードまたは低確信度がないか

**確認方法**:
1. JSON出力を確認
2. 各カテゴリの `observations` に出典が含まれているか
3. 各カテゴリの `confidence` を確認（「低」が3つ以上あるか）
4. 高スコア（80以上）のカテゴリで否定的キーワード（不安定、不明瞭、矛盾等）がないか

**判定**: 警告が0件の場合、ハルシネーション検証は **OK**

### 4.4 結果の妥当性確認（主観的評価）

- [ ] **観察事実の妥当性**: 動画の内容と観察事実が一致しているか（人間が動画を確認）
- [ ] **スコアの妥当性**: スコアが観察事実と矛盾していないか
- [ ] **推奨事項の妥当性**: 推奨事項が評価結果と整合しているか

**注意**: これは主観的な評価であり、統計的測定ではありません。明らかにおかしい場合のみ指摘してください。

**判定**: 明らかな不整合がない場合、結果の妥当性は **OK**

## 5. 合格判定基準

### 5.1 合格基準

以下の4項目を全て満たす場合、**限定的PoC合格**とします：

| 項目 | 基準 |
|---|---|
| **システム動作** | エラーなく解析完了、結果表示 |
| **出典品質** | 全ての観察事実に出典が含まれる |
| **ハルシネーション** | 警告0件（出典欠落なし、確信度低3つ未満、矛盾なし） |
| **結果の妥当性** | 明らかな不整合がない |

### 5.2 合格時の対応

**運用開始を承認**します。

**次のステップ**:
1. 運用マニュアルに従って運用開始
2. Phase 3（運用開始後）で統計的精度検証を実施
3. フィードバック収集と継続的改善

**参照**: `docs/operation_manual.md`

### 5.3 不合格時の対応

**運用開始を保留**し、以下の対応を実施します。

#### 不合格パターン1: システム動作エラー

**原因**:
- APIエラー
- 動画形式の問題
- ネットワークエラー

**対応**:
1. エラーメッセージを確認
2. 「運用マニュアル - トラブルシューティング」を参照
3. 技術サポートに報告
4. 問題解決後、再度PoC実施

#### 不合格パターン2: 出典欠落

**原因**:
- プロンプトで出典明記の指示が不十分
- AIが指示を無視

**対応**:
1. `src/prompts/system.txt` を確認
2. 出典明記の制約を強化
3. 再度PoC実施

#### 不合格パターン3: ハルシネーション警告あり

**原因**:
- AIが根拠のない推測をした
- ナレッジベースの範囲外の評価をした

**対応**:
1. 警告内容を確認
2. プロンプト・ナレッジベースを改善
3. 再度PoC実施

#### 不合格パターン4: 結果の妥当性に問題

**原因**:
- AIの評価が動画内容と一致しない
- スコアと観察事実が矛盾

**対応**:
1. 動画を再確認
2. プロンプト・ナレッジベースを改善
3. 別の動画で再度PoC実施

## 8. FAQ

### Q1: なぜ統計的精度検証をしないのですか？

**A**: 以下の理由により、限定的PoCに留めています：

1. **実動画が1件のみ**: 統計的測定には5〜10件必要
2. **人間評価データなし**: 相関係数を計算するには人間評価が必要
3. **運用開始を優先**: 基本機能が整っているため、動作確認のみで運用開始

Phase 3（運用開始後）で、5〜10名分のデータを収集して統計的精度検証を実施します。

### Q2: 限定的PoCで十分ですか？

**A**: 基本機能とハルシネーション対策が実装されているため、動作確認のみで運用開始は可能です。ただし、**精度は未検証**であることを理解し、以下の運用方針で進めてください：

- AIの評価は「参考情報」として扱う
- 最終判断は必ず人間が行う
- 運用開始後、Phase 3で精度検証を実施
- 継続的な改善を実施

### Q3: 不合格の場合はどうなりますか？

**A**: 運用開始を保留し、問題点を改善してから再度PoC実施します。詳細は「5.3 不合格時の対応」を参照してください。
