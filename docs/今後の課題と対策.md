## 1. PoC実施による精度検証 [計画変更 - Phase 3に延期]

**開始日**: 2026-03-16
**計画変更日**: 2026-03-16
**ステータス**: 延期（Phase 3で実施）

**変更理由**:
- 実動画1件のみ、人間評価なし（統計的検証不可能）
- 運用開始を優先（「すぐ」との要件）
- Iteration-07で限定的PoC（動作確認のみ）を実施後、運用開始
- Phase 3（運用開始後）で5-10件のデータが蓄積された時点で本格的な精度検証を実施

### 1.1 現状の問題

| 項目 | 現状 | 問題 |
|------|------|------|
| 実動画テスト | 1件のみ | サンプル数が不足 |
| 評価結果 | 95点（優秀） | 1件では統計的に不十分 |
| 人間評価との比較 | 未実施 | 相関が不明 |
| 偽陽性率 | 不明 | 問題ない人を誤判定する率 |
| 偽陰性率 | 不明 | 問題ある人を見逃す率 |
| ハルシネーション | 未検証 | もっともらしい嘘の有無 |

### 1.2 目標

| 指標 | 目標値 | 理由 |
|------|--------|------|
| **相関係数** | > 0.7 | 人間の評価と強い相関があることを示す |
| **偽陽性率** | < 20% | 5人に1人以下の誤判定 |
| **偽陰性率** | < 10% | 見逃しは致命的なのでより厳格に |
| **ハルシネーション** | 0件 | 虚偽の報告は許容できない |
| **サンプル数** | 5～10名 | 統計的に意味のある最小サンプル |

### 1.3 実施方法

#### ステップ1: 対象動画の選定（1日）

**選定基準**:
- 過去の面談動画（結果がわかっているもの）
- 多様なパターンを含む:
  - ✅ 優秀だった候補者（2～3名）
  - ⚠️ 問題が発生した候補者（2～3名）
  - 🤔 判断が難しかった候補者（1～2名）

**必要な情報**:
- 面接官の評価（当時の評価シート）
- アサイン結果（成功/問題発生/その他）
- 問題の種類（スキル不足/対人トラブル/途中辞退など）

#### ステップ2: AI解析の実行（1日）

```bash
# 各動画を解析
python src/analyzer.py video1.mp4 --output results/poc_candidate1.json
python src/analyzer.py video2.mp4 --output results/poc_candidate2.json
...

# 結果をスプレッドシートに転記
- 候補者ID
- AI総合スコア
- AI評価（コミュニケーション、ストレス耐性、信頼性、チームワーク）
- AIのリスクシグナル
- AIの推奨事項
```

#### ステップ3: 人間評価との比較（2日）

**比較項目**:

1. **総合スコアの相関**
   ```
   人間の評価（5段階）とAIスコア（0-100点）の散布図を作成
   相関係数を計算（目標: r > 0.7）
   ```

2. **リスクシグナルの一致率**
   ```
   AIが検出したリスク vs 人間が気になった点
   一致: ○
   AIのみ検出: △（偽陽性の可能性）
   人間のみ検出: ×（偽陰性の可能性）
   ```

3. **実際の結果との照合**
   ```
   AIの予測 vs 実際のアサイン結果
   - 高評価 → 成功: ✅ 正解
   - 高評価 → 問題発生: ❌ 偽陰性（見逃し）
   - 低評価 → 成功: ❌ 偽陽性（誤判定）
   - 低評価 → 問題発生: ✅ 正解
   ```

#### ステップ4: ハルシネーションの検証（1日）

**検証方法**:
- AIの観察事項を1つずつ動画で確認
- 実際にその行動が見られるか
- 存在しない行動を報告していないか

**記録フォーマット**:
```
候補者ID: 001
観察事項: "適切なアイコンタクトを維持"
動画時刻: 12:30-12:45
検証結果: ✅ 事実（アイコンタクトが確認できた）

候補者ID: 002
観察事項: "質問に対して即答しすぎる"
動画時刻: 全体
検証結果: ❌ ハルシネーション（実際は2-3秒考えてから回答）
```

#### ステップ5: 結果の分析と改善（2日）

**分析項目**:
1. 相関係数が0.7未満の場合 → 原因を特定
   - ナレッジベースが不十分？
   - システムプロンプトが曖昧？
   - 評価基準が現場と合っていない？

2. 偽陽性が多い場合 → 評価基準が厳しすぎる
   - ナレッジベースの「リスクシグナル」を見直し
   - 「確信度: 低」の項目を除外

3. 偽陰性が多い場合 → 見逃しが多い
   - ナレッジベースに新しいリスクパターンを追加
   - システムプロンプトを強化

4. ハルシネーションが発生した場合
   - 該当する評価項目を特定
   - プロンプトに「観察された事実のみ報告」を強調

**改善アクション**:
- `knowledge-base/core-criteria.md` の更新
- `src/prompts/system.txt` の調整
- 新しいテストケースの追加

- `results/poc_analysis.xlsx` - 詳細データ（相関分析、誤判定分析）

## 2. ハルシネーション対策の強化 [✅ 完了 - Iteration-04]

**開始日**: 2026-03-16
**完了日**: 2026-03-16
**ステータス**: 完了（93点、A評価）
**成果物**: `docs/evaluation_guidelines.md`, システムプロンプト更新, バリデーション実装

### 2.1 現状の問題

| 問題 | 影響 |
|------|------|
| **出典が不明確** | AIの評価がどのナレッジベースを参照したか不明 |
| **検証が困難** | ハルシネーション（虚偽）かどうかを判断できない |
| **確信度の扱い** | 「確信度: 低」の評価をどう扱うべきか未定義 |

### 2.2 目標

1. すべての評価項目に**出典**（ナレッジベースのセクション番号）を明記
2. ハルシネーション検出率 > 95%
3. 確信度が低い評価の取り扱いルールを策定

### 2.3 実施方法

#### 対策1: システムプロンプトの更新

**追加する制約**:
```
## 重要な制約（追加）
- **出典の明記**: 各評価項目について、ナレッジベースのどのセクション（1-8）を
  参照したかを必ず明記してください。

  例:
  "観察事項: 適切なアイコンタクトを維持
   参照: ナレッジベース セクション1「非言語コミュニケーション - 視線」"

- **観察された事実のみ**: 動画で実際に見えた・聞こえたことのみを報告し、
  推測や想像は一切含めないでください。

- **確信度の厳格な判定**: 確信度が「低」の場合は、その理由を明記してください。
  （例: 「確信度: 低（該当する行動が1回のみで、偶然の可能性）」）
```

#### 対策2: レスポンスパーサーのバリデーション強化

**バリデーションルール**:
1. 出典が明記されていない評価項目を警告
2. 確信度が低い項目が50%以上の場合に警告
3. 矛盾する観察事項を検出（例: 「アイコンタクトが多い」と「視線を避ける」が同時）

#### 対策3: 確信度が低い評価の取り扱いルール

**ルール**:
```
確信度: 高 → 評価に含める（重み100%）
確信度: 中 → 評価に含める（重み80%）
確信度: 低 → 参考情報として記載（重みなし）

確信度が低い項目が50%以上の場合:
→ 警告メッセージを表示し、人間による追加確認を推奨
```

## 3. 継続的な改善のフィードバックループ構築

### 3.1 現状の問題

| 問題 | 影響 |
|------|------|
| **フィードバック収集の仕組みがない** | 評価結果が適切だったか不明 |
| **実際の結果との照合なし** | AIの予測精度が測定できない |
| **改善が属人的** | プロンプトやナレッジベースの更新が場当たり的 |
| **精度低下の検知不可** | 運用後に精度が下がっても気づけない |

### 3.2 目標

1. 評価結果と実際のアサイン後の結果を**追跡**
2. **3ヶ月ごと**に精度を再評価
3. プロンプト・ナレッジベースの**定期的な改善**

### 3.3 実施方法

#### フェーズ1: データ収集の仕組み構築

**評価結果記録スプレッドシートの作成**:

| 列 | 内容 |
|---|---|
| 候補者ID | 匿名化されたID |
| 面談日 | YYYY-MM-DD |
| AI総合スコア | 0-100点 |
| 人間の評価 | 5段階評価 |
| アサイン結果 | 成功/問題発生/その他 |
| 問題の種類 | スキル不足/対人トラブル/途中辞退/なし |
| フィードバック | 自由記述 |

#### フェーズ2: 定期的な分析と改善

**3ヶ月ごとの精度分析（四半期レビュー）**:

1. 相関係数の推移
2. 偽陽性率・偽陰性率の推移
3. ハルシネーション発生率
4. カテゴリ別の精度

**改善のPDCAサイクル**:
```
[Plan] 四半期レビューで課題を特定
   ↓
[Do] ナレッジベース、プロンプトを更新
   ↓
[Check] 過去の動画で再テスト（精度向上を確認）
   ↓
[Action] 本番環境に反映、次回レビューで効果測定
```

#### フェーズ3: バックテスト（運用6ヶ月後）

**目的**: AIの長期的な予測精度を検証

**分析項目**:
1. AIの評価と6ヶ月後の結果の相関
2. ROIの再計算
3. 長期的な精度推移

### 3.4 成果物

- `results/evaluation_tracker.xlsx` - 評価結果記録スプレッドシート
- `results/quarterly_review_Q1.md` - 四半期レビューレポート（3ヶ月ごと）
- `results/backtest_6months.md` - バックテストレポート（6ヶ月後）
