# 評価ガイドライン（Iteration-04: ハルシネーション対策）

## 概要

本ドキュメントは、AI面談動画解析システムにおける評価の信頼性を担保するためのガイドラインです。特に、AIの「ハルシネーション（事実に基づかない情報の生成）」を防止するための基準と運用方針を定めます。

**作成日**: 2026-03-16
**バージョン**: 1.0
**対象イテレーション**: Iteration-04

---

## 1. 確信度の定義

AIは各評価カテゴリに対して、以下の3段階で確信度を判定します。

### 確信度「高」
- **定義**: 動画内で明確に観察でき、ナレッジベースの基準と明確に一致する
- **具体例**:
  - 明瞭な発話が繰り返し確認できる
  - アイコンタクトが全編を通じて安定している
  - 具体的なエピソードを複数回詳細に説明している
- **重み付け**: 評価の信頼性が高く、そのまま参考にできる

### 確信度「中」
- **定義**: 観察できるが、文脈や解釈に若干の余地がある
- **具体例**:
  - 発話は明瞭だが、一部聞き取りにくい箇所がある
  - アイコンタクトは概ね安定しているが、時折逸れる
  - エピソードはあるが、詳細が不足している
- **重み付け**: 評価の参考にできるが、他の情報と照合することが望ましい

### 確信度「低」
- **定義**: 情報が不十分、または判断に大きな不確実性がある
- **具体例**:
  - 動画の音質が悪く、発話内容が不明瞭
  - カメラアングルの問題で表情や視線が確認できない
  - 該当する質問が動画内になく、評価の根拠が乏しい
- **重み付け**: 評価の信頼性が低く、追加情報の取得が必要

## 2. 出典明記ルール

### 2.1 基本方針

**すべての観察事実には、参照したナレッジベースのセクション名を明記する**

これにより、以下を実現します:
- AIがナレッジベースに基づいて評価したことの証明
- ハルシネーション（根拠のない推測）の防止
- レビュー時のトレーサビリティ向上

### 2.2 出典形式

```
観察事実（参照: <ナレッジベースのセクション名>）
```

**例**:
- 「適切なアイコンタクトを維持（参照: 非言語コミュニケーション基準）」
- 「具体的なエピソードを交えて説明（参照: 信頼性評価基準）」
- 「難しい質問にも落ち着いて対応（参照: ストレス耐性評価基準）」

### 2.3 出典が明記されていない場合の取り扱い

システムは自動的に以下の警告を生成します:

```
警告: 出典が欠落している観察事実が検出されました。
カテゴリ: 'communication'
観察事実: '適切なアイコンタクト'
```

**運用上の対応**:
1. 警告が出た場合、その評価結果の信頼性を疑う
2. 人間のレビュアーが動画を確認し、観察事実が正しいか検証する
3. 必要に応じて、AIに再評価を依頼する

## 3. 確信度閾値ルール

### 3.1 閾値の定義

**4つの評価カテゴリ（communication, stress_tolerance, reliability, teamwork）のうち、3つ以上が確信度「低」の場合、警告を出す**

これは、評価全体の信頼性が疑わしいことを示します。

### 3.2 警告メッセージ

```
警告: 確信度が低い評価が多すぎます: 4/4カテゴリが確信度「低」。
評価の信頼性に問題がある可能性があります。
```

### 3.3 確信度「低」が多い場合の取り扱い

| 確信度「低」の数 | 対応方針 |
|---|---|
| 0-2個 | 通常通り評価結果を参考にする |
| 3個以上 | **警告**: 動画の品質不足、または評価が困難 |
| 4個（全て） | **推奨**: 追加面談を実施、または動画を再撮影 |

**根本原因の例**:
- 動画の音質・画質が不十分
- 面談時間が短すぎて評価材料が不足
- 候補者が極端に寡黙で、観察できる情報が少ない
- AIのモデルが該当シーンを正しく理解できなかった

**運用上の対応**:
1. 動画の品質（音質、画質、時間）を確認
2. 面談の質（質問の種類、候補者の回答量）を評価
3. 必要に応じて追加面談を実施
4. システム側の問題（モデル、プロンプト）の可能性も検討

## 4. 矛盾検出ルール

### 4.1 矛盾の定義

AIの評価に以下のような矛盾がある場合、ハルシネーションの兆候と見なします。

#### 矛盾パターン1: スコアと観察事実の不一致

**定義**: スコアが80以上（高評価）なのに、観察事実に否定的なキーワードが含まれる

**否定的キーワードの例**:
- 不安定、不明瞭、矛盾、避ける、批判的、回避
- 不十分、欠如、問題、困難、弱い、乏しい

**例**:
```json
{
  "communication": {
    "score": 90,  // 高スコア
    "observations": [
      "視線が不安定で回避的（参照: 非言語コミュニケーション基準）",  // 否定的
      "発話が不明瞭で聞き取りにくい（参照: 言語表現評価基準）"  // 否定的
    ],
    "confidence": "低"
  }
}
```

**警告メッセージ**:
```
警告: 矛盾を検出: カテゴリ'communication'はスコア90（高評価）ですが、
否定的な観察事実が含まれています: '視線が不安定で回避的...'
```

#### 矛盾パターン2: スコアと確信度の不一致

**定義**: スコアが80以上（高評価）なのに、確信度が「低」

**例**:
```json
{
  "reliability": {
    "score": 95,  // 非常に高いスコア
    "observations": ["エピソードの詳細が不明（参照: 信頼性評価基準）"],
    "confidence": "低"  // 低確信度
  }
}
```

**警告メッセージ**:
```
警告: 矛盾を検出: カテゴリ'reliability'はスコア95（高評価）ですが、
確信度が「低」となっています。評価の整合性を確認してください。
```

### 4.2 矛盾が検出された場合の取り扱い

**運用上の対応**:
1. **重要度: 高** - 矛盾は重大なハルシネーションの兆候
2. その評価結果を信頼せず、人間が動画を再確認する
3. AIに再評価を依頼し、矛盾が解消されるか確認
4. 繰り返し矛盾が発生する場合、プロンプトやナレッジベースの改善が必要

## 5. バリデーション機能の実装

### 5.1 実装場所

`src/response_parser.py` の `validate_response()` 関数

### 5.2 使用方法

```python
from src.response_parser import parse_response, validate_response

# AIレスポンスをパース
response_data = parse_response(ai_response_text)

# ハルシネーション対策のバリデーション
warnings = validate_response(response_data)

# 警告の確認
if warnings:
    print("⚠️ 評価に以下の問題が検出されました:")
    for warning in warnings:
        print(f"  - {warning}")
else:
    print("✅ 評価に問題は検出されませんでした")
```

### 5.3 戻り値

- **型**: `list[str]`
- **内容**: 警告メッセージのリスト
- **空リスト**: 警告なし（評価に問題なし）

## 6. 運用フロー

### 6.1 通常フロー（警告なし）

```
1. AIが動画を解析
   ↓
2. レスポンスをパース（parse_response）
   ↓
3. バリデーション実行（validate_response）
   ↓
4. 警告なし → 評価結果を採用
   ↓
5. 人間が最終判断
```

### 6.2 警告ありフロー

```
1. AIが動画を解析
   ↓
2. レスポンスをパース（parse_response）
   ↓
3. バリデーション実行（validate_response）
   ↓
4. 警告あり → 警告内容を確認
   ↓
5. 出典欠落の警告
   → ナレッジベースとの整合性を確認
   → 必要に応じて再評価
   ↓
6. 確信度低の警告
   → 動画品質を確認
   → 追加面談を検討
   ↓
7. 矛盾検出の警告
   → 重要度: 高
   → 人間が動画を再確認
   → 評価結果を信頼しない
   ↓
8. 人間が最終判断
```

## 7. 制約と仮定

### 7.1 仮定

1. **出典形式**: 「(参照: <セクション名>)」または「（参照: <セクション名>）」
2. **確信度閾値**: 4カテゴリ中3つ以上が「低」で警告
3. **矛盾の定義**:
   - スコア80以上 + 否定的キーワード
   - スコア80以上 + 確信度「低」

### 7.2 制約

1. **バリデーションの限界**: 機械的なチェックのため、文脈理解が不十分な場合がある
2. **偽陽性の可能性**: 正常な評価に対して誤って警告が出る可能性がある
3. **否定的キーワードの網羅性**: すべての否定的表現を網羅できていない可能性がある

### 7.3 今後の改善案

- [ ] 自然言語処理による文脈理解の向上
- [ ] 否定的キーワードリストの拡充
- [ ] 矛盾検出の閾値の最適化（スコア80以上が適切か検証）
- [ ] 統計的手法による異常検出（スコアの分散、外れ値検出）
