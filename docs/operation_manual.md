# AI面談動画解析システム - 運用マニュアル

**作成日**: 2026-03-16
**バージョン**: 1.0
**対象**: 採用担当者、面接官、技術サポート担当者

---

## 1. システム概要

### 1.1 システムの目的

AI面談動画解析システムは、SES事業における面談の品質向上と客観的な評価を支援するシステムです。

**主な機能**:
- 面談動画のAI解析
- 行動心理学に基づく評価レポート生成
- ハルシネーション（事実に基づかない推測）の検出

**重要**: 本システムは採用判断の「参考情報」を提供するものであり、最終判断は必ず人間が行います。

### 1.2 システム構成

- **フロントエンド**: Streamlit Web UI（ブラウザで操作）
- **バックエンド**: Python + Gemini AI（Google製）
- **データ保管**: ローカル/クラウドストレージ

### 1.3 精度と限界

**現在の状態（Phase 2完了時点）**:
- ✅ 基本機能実装済み
- ✅ ハルシネーション対策済み
- ⚠️ **統計的精度検証は未実施**（運用開始後にPhase 3で実施）

**制約事項**:
- 実動画での大規模検証は未実施
- 偽陽性率・偽陰性率は暫定的な推定値
- 運用開始後、継続的な精度改善が必要

## 2. 運用フロー

### 2.1 全体の流れ

```
1. 事前準備
   ↓
2. 候補者への説明と同意取得
   ↓
3. 面談実施（動画録画）
   ↓
4. 動画準備（ファイル確認）
   ↓
5. システムへアップロード
   ↓
6. AI解析実行（数分〜10分）
   ↓
7. 結果確認（評価レポート）
   ↓
8. 人間による最終判断
   ↓
9. データ保管/廃棄
```

### 2.2 各ステップの詳細

#### ステップ1: 事前準備

**実施内容**:
- [ ] Streamlit UIを起動
- [ ] APIキーが設定されていることを確認
- [ ] ネットワーク接続を確認

**コマンド**:
```bash
# Streamlit UIの起動
cd ai_interview-analysis
streamlit run src/streamlit_app.py
```

#### ステップ2: 候補者への説明と同意取得

**実施内容**:
- [ ] 候補者に「AI面談動画解析システム - 候補者同意書」を提示
- [ ] システムの目的、利用範囲、AI解析の仕組みを説明
- [ ] 同意の任意性と撤回権を説明
- [ ] 同意書に署名をもらう（紙またはデジタル）

**注意事項**:
- 同意しない場合でも採用選考から除外しない
- 強制や圧力は厳禁
- 不明点があれば丁寧に説明

**参照**: `docs/candidate_consent_form.md`

#### ステップ3: 面談実施（動画録画）

**推奨設定**:
- **時間**: 10〜30分程度
- **形式**: MP4（H.264エンコード）
- **解像度**: 720p以上
- **音質**: クリアな音声（ノイズが少ない）
- **カメラアングル**: 候補者の表情・視線が確認できる

**録画方法**:
- Zoom, Teams, Google Meetの録画機能
- 専用カメラでの録画

**注意事項**:
- 候補者に録画していることを事前に伝える
- 録画開始前に同意を再確認
- 録画中は「録画中」の表示を明示

#### ステップ4: 動画準備（ファイル確認）

**確認項目**:
- [ ] ファイル形式: MP4, MOV, AVI, WebM
- [ ] ファイルサイズ: 500MB以下（推奨）
- [ ] 動画時間: 10〜30分程度
- [ ] 画質・音質: 表情・音声が明瞭

**問題がある場合**:
- ファイル形式が非対応 → MP4に変換
- ファイルサイズが大きすぎる → 圧縮またはチャンク分割
- 画質・音質が悪い → 再録画を検討

#### ステップ5: システムへアップロード

**Streamlit UIでの操作**:

1. ブラウザで `http://localhost:8501` を開く
2. サイドバーで「Gemini APIキー」を入力
3. 「面談動画をアップロード」ボタンをクリック
4. 動画ファイルを選択
5. ファイル情報（ファイル名、サイズ、形式）を確認

**注意事項**:
- アップロード中はブラウザを閉じない
- ネットワーク接続が安定していることを確認
- APIキーは毎回入力（セキュリティ上、保存しない）

#### ステップ6: AI解析実行

**Streamlit UIでの操作**:

1. 「解析開始」ボタンをクリック
2. 「動画を解析中...」のメッセージが表示される
3. 数分〜10分待機（動画の長さによる）

**処理時間の目安**:
- 10分動画: 約3〜5分
- 30分動画: 約5〜10分

**注意事項**:
- 解析中はブラウザを閉じない
- エラーが発生した場合は、エラーメッセージを確認（後述のトラブルシューティング参照）

#### ステップ7: 結果確認（評価レポート）

**Streamlit UIの表示内容**:

1. **総合評価**:
   - 総合スコア（0〜100）
   - リスクレベル（低/中/高）

2. **詳細評価**（タブ表示）:
   - コミュニケーション
   - ストレス耐性
   - 信頼性
   - 協調性

3. **リスクシグナル**:
   - 懸念事項のリスト

4. **ポジティブシグナル**:
   - 優れている点のリスト

5. **推奨事項**:
   - AIによる採用推奨の有無

6. **詳細なJSON出力**（折りたたみ）:
   - 完全な評価結果のJSON

**確認ポイント**:
- ✅ ハルシネーション警告が出ていないか
- ✅ 観察事実に出典（参照: ナレッジベース）が含まれているか
- ✅ スコアと観察事実に矛盾がないか

#### ステップ8: 人間による最終判断

**判断のポイント**:

1. **AIの評価を参考にする**:
   - 総合スコア、リスクシグナルを確認
   - 観察事実が妥当か検証

2. **自身の印象と照合**:
   - 面談時の印象とAI評価を比較
   - 矛盾がある場合は、理由を考える

3. **エスカレーション基準**（後述）:
   - ハルシネーション警告が出た場合
   - スコアと観察事実に大きな矛盾がある場合
   - 判断に迷った場合

4. **最終判断**:
   - 採用/見送り/追加面談
   - 判断理由を記録

**重要**: AIの評価は参考情報であり、最終判断は人間が行います。AIの評価に反する判断も可能です。

#### ステップ9: データ保管/廃棄

**保管ルール**:
- **採用された場合**: 契約期間中保管（最長3年）
- **不採用の場合**: 6ヶ月保管後、廃棄

**廃棄手順**:
1. 保管期間終了日をカレンダーに記録
2. 期間終了後、動画ファイルとAI評価結果を削除
3. 廃棄記録を残す（候補者ID、廃棄日時）

## 4. トラブルシューティング

### 4.1 よくあるエラーと対処法

#### エラー1: 「APIキーを入力してください」

**原因**: APIキーが未入力

**対処法**:
1. サイドバーの「Gemini APIキー」欄に正しいAPIキーを入力
2. APIキーの取得方法: [Google AI Studio](https://aistudio.google.com/app/apikey)

#### エラー2: 「Video file not found」

**原因**: 動画ファイルが見つからない

**対処法**:
1. ファイルパスが正しいか確認
2. ファイルが存在するか確認
3. ファイル名に特殊文字が含まれていないか確認

#### エラー3: 「Unsupported video format」

**原因**: サポート外の動画形式

**対処法**:
1. ファイル形式を確認（対応形式: MP4, MOV, AVI, WebM）
2. MP4形式に変換（VLC, HandBrake等のツールを使用）

#### エラー4: 「Failed to parse JSON」

**原因**: AIのレスポンスがJSON形式でない

**対処法**:
1. 再度解析を実行
2. 繰り返し発生する場合は、技術サポートにエスカレーション

#### エラー5: 「ハルシネーション警告が出た」

**原因**: AIが根拠のない推測をした可能性

**対処法**:
1. 評価結果を慎重に確認
2. 観察事実に出典（参照: ナレッジベース）が含まれているか確認
3. 矛盾がある場合は、その評価を信頼しない
4. 技術サポートにエスカレーション

### 4.2 システムエラーの確認方法

**ログの確認**:
```bash
# Streamlitのログを確認
# ターミナルに表示されるエラーメッセージを確認
```

**エラーメッセージの記録**:
- エラーメッセージをスクリーンショットまたはコピー
- 発生日時、実施内容、動画ファイル名を記録
- 技術サポートに報告

## 5. エスカレーション基準

以下の場合は、技術サポートまたは上長にエスカレーションしてください：

### 5.1 ハルシネーション警告が出た場合

**判断基準**:
- 観察事実に出典（参照: ナレッジベース）が欠落
- スコアと観察事実に大きな矛盾
- 確信度「低」が3つ以上

**対応**:
1. その評価結果を信頼しない
2. 人間の判断を優先
3. 技術サポートに報告（プロンプト・ナレッジベースの改善が必要）

### 5.2 AIの評価が不自然な場合

**判断基準**:
- スコアが極端（0点または100点）
- 観察事実が曖昧または抽象的
- 推奨事項が妥当でない

**対応**:
1. 動画を再確認
2. 人間の判断を優先
3. 技術サポートに報告

### 5.3 判断に迷った場合

**判断基準**:
- AIの評価と自身の印象が大きく異なる
- ボーダーライン（スコア60〜70）で判断が難しい
- 追加情報が必要

**対応**:
1. 上長に相談
2. 追加面談を実施
3. AIの評価は参考程度に扱う

## 6. 運用開始後の改善計画

### 6.1 Phase 3: 統計的精度検証（運用開始後）

**目的**: AIの評価精度を定量的に測定

**実施内容**:
1. 運用開始後、5〜10名分のデータを収集
2. AI評価と人間評価の相関係数を計算
3. 偽陽性率・偽陰性率を測定
4. 合格基準（相関係数 > 0.7、FPR < 20%、FNR < 10%）を満たすか確認

**実施時期**: 運用開始から3ヶ月後

**参照**: `docs/poc_plan.md`, `docs/poc_execution_guide.md`

### 6.2 フィードバック収集

**収集する情報**:
- AI評価の正確性（人間の評価と一致したか）
- ハルシネーション警告の有無
- 運用上の課題（操作性、処理時間等）

**収集方法**:
- 月次レビュー会議で報告
- アンケート（採用担当者向け）

### 6.3 継続的な改善

**改善サイクル**:
```
1. フィードバック収集
   ↓
2. 問題点の特定
   ↓
3. プロンプト・ナレッジベースの改善
   ↓
4. テスト実施
   ↓
5. 運用環境への反映
   ↓
6. 精度検証
   ↓
1に戻る（3ヶ月ごと）
```

## 7. FAQ（よくある質問）

### Q1: AIの評価は信頼できますか？

**A**: 本システムは基本機能が実装され、ハルシネーション対策も施されていますが、**統計的精度検証は未実施**です。運用開始後、Phase 3で精度検証を行います。現時点では、AIの評価は「参考情報」として扱い、最終判断は必ず人間が行ってください。

### Q2: 候補者が同意しない場合はどうなりますか？

**A**: 同意しない場合でも、採用選考から除外されることはありません。従来通りの人間による面談評価のみで選考を行います。

### Q3: AIの評価に反する判断をしても良いですか？

**A**: はい、最終判断は人間が行うため、AIの評価に反する判断も可能です。ただし、判断理由を記録してください。

### Q4: 動画はどこに保管されますか？

**A**: ローカルストレージまたはセキュアなクラウドストレージ（Google Cloud Storage等）に保管されます。アクセス制限があり、採用担当者および技術担当者のみがアクセス可能です。

### Q5: 処理時間はどのくらいですか？

**A**: 10分動画で約3〜5分、30分動画で約5〜10分程度です。動画の長さやネットワーク状況により変動します。

### Q6: エラーが発生した場合はどうすればよいですか？

**A**: 本マニュアルの「4. トラブルシューティング」を参照してください。解決しない場合は、技術サポートにエスカレーションしてください。
