# 倫理・コンプライアンス検討事項

## 本施策における重大なリスクと懸念点

技術的には実現可能ですが、**実際の運用にあたっては以下のリスクをクリアにする必要があります**。

---

## 1. 候補者の同意とプライバシー保護

### 法的要件
- **個人情報保護法**: 面談動画は個人情報（顔、声、行動パターン等）に該当
- **特定個人情報**: 心理的プロファイリングは「要配慮個人情報」に該当する可能性

### 必須対応事項

#### 事前同意の取得
面談動画をAIに解析させること、また心理的プロファイリングを行うことについて、**候補者からの明示的な事前同意が必須**です。

**同意なき解析は重大なコンプライアンス違反**となります。

#### 同意取得のポイント
- **書面またはオンラインフォームでの明示的同意**
- 以下の内容を明確に説明し、同意を得る:
  - 動画が録画されること
  - AIによる解析が行われること
  - 解析の目的（採用選考における参考情報として使用）
  - データの保存期間と管理方法
  - データへのアクセス権限を持つ者
  - 同意を拒否する権利と、拒否による不利益がないこと

#### 推奨される同意文例
```
【AI動画解析に関する同意書】

当社は、採用選考の精度向上を目的として、
面談動画をAI（人工知能）で解析し、
行動心理学的な観点から参考情報を取得しております。

本解析は、あくまで人間による選考判断を補完する
参考情報として活用するものであり、
AI解析結果のみで合否を決定することはありません。

以下の内容にご同意いただける場合のみ、
チェックボックスにチェックをお願いいたします。

□ 面談動画が録画されることに同意します
□ 録画された動画がAIで解析されることに同意します
□ 解析結果が採用選考の参考情報として使用されることに同意します
□ データの保管期間は選考終了後6ヶ月以内であることを理解しました
□ 同意を拒否する権利があり、拒否しても不利益がないことを理解しました

署名:________________  日付:________________
```

## 2. AIのハルシネーションとバイアス

### リスクの本質
AIが「この人物は行動心理学的にトラブルを起こす確率が80%です」と出力しても、**それは確率論であり事実ではありません**。

### 主なリスク要因

#### ① 偽陽性（False Positive）
- 緊張による視線の泳ぎを「虚偽の兆候」と誤判定
- 文化的背景による行動パターンの違いを「リスク」と誤認
- 個人の特性（発達障害、内向的性格等）を「問題行動」と判定

#### ② ハルシネーション
- AIが存在しない事実を「もっともらしく」生成
- 根拠のない推論を確信的に提示
- 特定のキーワードに過剰反応

#### ③ バイアスの増幅
- 学習データに含まれる社会的偏見の反映
- 特定の属性（年齢、性別、外見等）に基づく不当な評価
- マイノリティに不利な判定傾向

### 必須対応事項

#### 運用ルールの確立
**AIの判定を「参考情報」にとどめ、最終決定は必ず人間が行う運用フローが不可欠**

```
【AI解析結果の取り扱いルール】

1. AI解析結果は「参考情報」であり、決定的な判断材料としない
2. 最終的な採用判断は、複数の人間（面接官）による協議で行う
3. AIが高リスクと判定した場合でも、必ず人間が根拠を検証する
4. AIの判定理由が不明確な場合は、その結果を採用しない
5. AI解析結果のみを理由とした不採用は禁止
```

#### バイアス対策
- **定期的な結果検証**: 実際の採用結果とAI判定の相関を分析
- **多様性の確保**: 特定の属性で不当に不利な判定が出ていないかモニタリング
- **説明可能性の確保**: AIがなぜその判定を下したのか、根拠を明示

## 3. データセキュリティとガバナンス

### データ保護要件

#### 動画データの取り扱い
- **暗号化**: 保存時・転送時ともに暗号化必須
- **アクセス制限**: 限定された担当者のみがアクセス可能
- **保存期間**: 必要最小限の期間（例: 選考終了後6ヶ月）で削除
- **バックアップ管理**: バックアップデータも同様に厳格管理

#### AI解析結果の管理
- **アクセスログ**: 誰がいつ結果を閲覧したか記録
- **改ざん防止**: 解析結果の後からの改変を防止
- **監査証跡**: コンプライアンス監査に対応可能な記録保持

### Gemini APIのデータポリシー理解
- **Google AI Studio**: 入力データがモデルの再学習に使用される可能性
- **Vertex AI**: エンタープライズ向けで、データは学習に使用されない（要確認）
- **選択すべきは**: セキュリティ要件に応じてVertex AIを選択

## 4. 透明性と説明責任

### 候補者への開示
- AI解析を実施していることを隠さない
- どのような観点で評価されるかを説明
- 結果に対する異議申し立ての仕組み

### 社内ガバナンス
- AI活用のガイドライン策定
- 定期的な倫理レビュー
- 問題発生時のエスカレーションフロー

### 第三者評価
- 必要に応じて外部の倫理委員会や専門家の助言を受ける
- 業界標準やベストプラクティスへの準拠

## まとめ

### 実現可能性と前提条件
**技術的には実現可能**ですが、以下の前提条件を満たすことが絶対条件:

1. **候補者からの明示的な事前同意**の取得
2. **AIの判定を参考情報とし、最終決定は人間が行う**運用ルール
3. **データセキュリティとプライバシー保護**の徹底
4. **透明性と説明責任**の確保

### リスクマネジメント
- 倫理的・法的リスクを軽視すると、企業の信用失墜や訴訟リスクに直結
- 慎重な準備と適切な運用体制の構築が成功の鍵

### 推奨アプローチ
1. **Phase 1**: 小規模PoC（社内同意者のみで検証）
2. **Phase 2**: 法務・倫理面の整備完了後、限定的な本番運用
3. **Phase 3**: 運用実績とフィードバックを基に、段階的に拡大

**「技術で実現できる」と「倫理的・法的に実施して良い」は別問題**という認識のもと、慎重かつ責任ある導入を進めることが重要です。
