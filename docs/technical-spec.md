# 技術仕様書

## システム概要

面談動画と行動心理学の専門知識を組み合わせ、Gemini 1.5 Pro APIを活用して候補者の行動リスクを評価するシステム。

---

## 技術スタック

### AIモデル
- **モデル名**: Gemini 1.5 Pro
- **提供元**: Google (AI Studio / Vertex AI)
- **選定理由**:
  - 長時間動画の処理能力（最大128kトークン以上のコンテキスト）
  - マルチモーダル対応（動画+テキストの同時処理）
  - 高精度な自然言語理解と生成

### インフラ候補

#### パターンA: Google Apps Script（簡易版）
- **実行環境**: Google Apps Script
- **ストレージ**: Google Drive
- **出力先**: Google Sheets / PDF
- **メリット**: ノーコード、初期コストゼロ、迅速な導入
- **デメリット**: スケーラビリティに制約、カスタマイズ性が低い

#### パターンB: GCP統合（本格版）
- **実行環境**: Cloud Functions (Node.js / Python)
- **ストレージ**: Cloud Storage
- **データベース**: BigQuery（オプション）
- **メリット**: 高スケーラビリティ、既存システムとの統合容易
- **デメリット**: 初期構築コスト、技術的専門知識が必要

---

## データフロー

### 全体フロー
```
[面談動画] ──┐
             ├─→ [Gemini API] ──→ [リスク評価レポート] ──→ [人間による判断]
[専門書]  ──┘
```

### 詳細処理ステップ

#### 1. 入力データの準備
- **面談動画**:
  - 形式: MP4, MOV, AVI等（Gemini対応形式）
  - 長さ: 30分/本
  - 解像度: 制約なし（APIが自動最適化）

- **専門書（ナレッジベース）**:
  - 形式: テキスト、PDF、Markdown
  - サイズ: 約10万トークン/書籍
  - 内容: 行動心理学、面接評価基準、トラブル事例等

- **システムプロンプト**:
  - 評価基準の明示
  - 出力フォーマットの指定
  - バイアス軽減のための指示

#### 2. API呼び出し
```python
# 疑似コード例
import google.generativeai as genai

# APIキー設定
genai.configure(api_key="YOUR_API_KEY")

# モデル選択
model = genai.GenerativeModel('gemini-1.5-pro')

# 入力データ準備
video_file = genai.upload_file(path="interview_video.mp4")
knowledge_text = read_file("psychology_knowledge.txt")

# プロンプト構築
prompt = f"""
あなたは行動心理学の専門家です。
以下のナレッジベースを基準として、面談動画を解析し、
候補者の行動リスクを評価してください。

【ナレッジベース】
{knowledge_text}

【評価項目】
1. コミュニケーション能力
2. ストレス耐性
3. 信頼性・誠実性
4. 協調性
5. その他の特記事項

【出力形式】
- 総合リスクスコア: 0-100
- 各評価項目のスコアと根拠
- 推奨される対応策
"""

# API呼び出し
response = model.generate_content([prompt, video_file])

# 結果取得
print(response.text)
```

## システム構成案

### パターンA: Google Apps Script版

#### 構成図
```
[Google Drive]
    ├── 面談動画フォルダ
    ├── ナレッジベースフォルダ
    └── 評価レポートフォルダ

[Google Apps Script]
    ├── main.gs (メイン処理)
    ├── gemini-api.gs (API呼び出し)
    └── report-generator.gs (レポート生成)

[Google Sheets]
    └── 評価結果一覧
```

#### 処理フロー
1. 担当者がGoogleドライブに面談動画をアップロード
2. GASのトリガー（時間ベース or 手動実行）で処理開始
3. Gemini APIに動画とナレッジを送信
4. 結果をGoogle Sheetsに記録、PDFレポートを生成
5. 担当者にメール通知

#### メリット
- **即座に導入可能**: コーディング最小限、Google Workspaceの既存環境で動作
- **コストゼロ**: API利用料以外の追加コストなし
- **運用が簡単**: 非エンジニアでも操作可能

#### デメリット
- **スケーラビリティ**: GASの実行時間制限（6分/回）に注意
- **カスタマイズ性**: 高度な処理は困難

---

### パターンB: GCP統合版

#### 構成図
```
[Cloud Storage]
    └── 面談動画バケット

[Cloud Functions]
    └── video-analysis-function
         ├── Gemini API呼び出し
         └── 結果をBigQueryに保存

[BigQuery]
    └── 評価結果データセット
         ├── candidates (候補者マスタ)
         └── evaluations (評価結果)

[Cloud Scheduler]
    └── 定期実行トリガー

[Cloud Logging]
    └── 処理ログ・エラーログ
```

#### 処理フロー
1. 採用システムから動画がCloud Storageにアップロード
2. Cloud StorageのトリガーでCloud Functions起動
3. Gemini APIに動画とナレッジを送信
4. 結果をBigQueryに保存
5. 必要に応じてSlack/メールで通知

#### メリット
- **高スケーラビリティ**: 処理件数増加に自動対応
- **既存システム統合**: APIで他システムと連携容易
- **データ分析**: BigQueryで過去データの分析・可視化が可能

#### デメリット
- **初期構築コスト**: インフラ設計・実装に時間と専門知識が必要
- **運用コスト**: インフラ費用が追加で発生

## セキュリティ仕様

### データ暗号化
- **転送時**: HTTPS/TLS 1.3以上
- **保存時**: Google Cloud KMSによる暗号化

### アクセス制御
- **IAM**: 最小権限の原則に基づく権限設定
- **認証**: Google Workspaceアカウント / Cloud Identity

### ログとモニタリング
- **アクセスログ**: Cloud Loggingで全アクセスを記録
- **異常検知**: Cloud Monitoringでアラート設定

### データ保持ポリシー
- **動画データ**: 選考終了後6ヶ月で自動削除
- **評価結果**: 選考終了後1年保持（法的要件に応じて調整）

## パフォーマンス仕様

### 処理時間
- **想定**: 1本の動画（30分）あたり 3〜5分で解析完了
- **並列処理**: GCP版では複数動画の同時処理可能

### スループット
- **月間50本**: 問題なく処理可能
- **将来の拡張**: 月間500本まで対応可能（GCP版）

## エラーハンドリング

### APIエラー
- **リトライ**: 一時的なエラーは3回まで自動リトライ
- **タイムアウト**: 10分でタイムアウト、管理者に通知

### データエラー
- **動画形式不正**: エラーログに記録、担当者に通知
- **ナレッジ読み込み失敗**: デフォルトのナレッジベースを使用
