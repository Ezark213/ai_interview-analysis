# PoC実施計画（Iteration-05）

## 概要

**目的**: AI面談動画解析システムの精度を実動画で検証し、実運用への適用可否を判断する

**期間**: 2026年3月中旬〜4月中旬（約1ヶ月）

**対象**: 過去の面談動画5〜10名分

**作成日**: 2026-03-16
**バージョン**: 1.0

---

## 1. PoC の目的

### 1.1 主要目標

1. **精度検証**: AIと人間の評価の一致度を定量的に測定
2. **ハルシネーション検出**: AIが事実に基づかない情報を生成していないか検証
3. **運用可能性の確認**: 実務で使用可能なレベルの精度か判断

### 1.2 合格基準

| 指標 | 閾値 | 理由 |
|---|---|---|
| **相関係数（Pearson）** | ≥ 0.7 | AIと人間の評価がよく一致していることを示す |
| **偽陽性率（FPR）** | < 20% | 問題ある候補者の20%未満しか見逃さない |
| **偽陰性率（FNR）** | < 10% | 優秀な候補者の10%未満しか誤って見送らない |
| **ハルシネーション警告** | 0件 | 根拠のない推測をしていない |

**総合判定**: 上記4つの基準を全て満たす場合、PoC合格とする

## 2. 対象動画の選定基準

### 2.1 選定方針

**バランスの取れたサンプル**: 優秀な候補者、問題ある候補者、判断困難な候補者をバランスよく含める

### 2.2 候補者の分類

| 分類 | 選定数 | 選定基準 |
|---|---|---|
| **優秀** | 2〜3名 | アサイン後に高いパフォーマンスを発揮 |
| **問題あり** | 2〜3名 | アサイン後にトラブル、または面談時に見送り |
| **判断困難** | 1〜2名 | 標準的なパフォーマンス、または評価が分かれた |

**合計**: 5〜8名

### 2.3 具体的な選定基準

#### 優秀な候補者の基準
- アサイン後、プロジェクトで高い成果を上げている
- 技術力・コミュニケーション力・協調性が優れている
- トラブルなく契約継続中、または円満な契約終了

#### 問題ある候補者の基準
- アサイン後、以下のいずれかのトラブルが発生:
  - 早期退職（3ヶ月以内）
  - 人間関係のトラブル
  - 技術力不足によるパフォーマンス問題
  - 契約解除
- または、面談時に見送りを決定し、実際にトラブルを避けられた

#### 判断困難な候補者の基準
- 標準的なパフォーマンス（特に優秀でもなく、問題もない）
- 面接官の評価が分かれた（採用を迷った）
- 採用後、特筆すべき成果もトラブルもない

## 5. 評価基準の詳細

### 5.1 相関係数（Pearson）

**計算方法**: AI評価スコア（`overall_risk_score`）と人間評価スコア（`human_score`）のPearson相関係数

**解釈**:
- r ≥ 0.7: 高い相関（AIの評価が信頼できる）
- 0.4 ≤ r < 0.7: 中程度の相関（改善の余地あり）
- r < 0.4: 低い相関（AIの精度に問題あり）

**合格基準**: r ≥ 0.7

### 5.2 偽陽性率（False Positive Rate）

**定義**: 問題ある候補者のうち、AIが誤って「採用推奨」した割合

**計算方法**: FPR = FP / (FP + TN)

**混同行列**:
- **FP（False Positive）**: AIが「採用推奨」したが、実際には「失敗」
- **TN（True Negative）**: AIが「見送り推奨」で、実際にも「失敗」

**合格基準**: FPR < 20%

**意味**: 20%未満であれば、問題ある候補者の80%以上を正しく検出できている

### 5.3 偽陰性率（False Negative Rate）

**定義**: 優秀な候補者のうち、AIが誤って「見送り推奨」した割合

**計算方法**: FNR = FN / (FN + TP)

**混同行列**:
- **FN（False Negative）**: AIが「見送り推奨」したが、実際には「成功」
- **TP（True Positive）**: AIが「採用推奨」で、実際にも「成功」

**合格基準**: FNR < 10%

**意味**: 10%未満であれば、優秀な候補者の90%以上を正しく推奨できている

### 5.4 ハルシネーション検証

**検証項目**:
1. **出典チェック**: 観察事実に「(参照: ナレッジベース)」が含まれるか
2. **確信度チェック**: 確信度「低」が3つ以上あるか
3. **矛盾チェック**: スコア80以上 + 否定的キーワード or 低確信度

**合格基準**: 警告0件

## 7. リスクと制約

### 7.1 リスク

1. **サンプルサイズが小さい**: 5〜10名では統計的に十分でない可能性
   - **対策**: 可能な限り10名に近づける、追加PoCを検討

2. **バイアスのあるサンプル**: 選定に偏りがある可能性
   - **対策**: 無作為抽出を心がける、複数の面接官の評価を含める

3. **動画品質のばらつき**: 過去の動画は品質が一定でない可能性
   - **対策**: 品質が低い動画は除外、または品質別に分析

### 7.2 制約

1. **候補者の同意**: 過去の動画を使用する場合、事後的な同意取得が必要
2. **個人情報保護**: 候補者名は匿名化、動画は厳重に管理
3. **時間的制約**: 1ヶ月以内に完了する必要がある
